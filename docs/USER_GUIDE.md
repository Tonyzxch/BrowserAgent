## 快速开始

### 安装步骤

1. **获取插件**
   - Docker部署：访问 http://localhost:8080 下载
   - 本地构建：运行 `npm run build && npm run pack`

2. **加载到Chrome**
   - 打开 `chrome://extensions/`
   - 开启"开发者模式"
   - 点击"加载已解压的扩展程序"
   - 选择 `dist` 目录

3. **首次使用**
   - 点击工具栏中的插件图标
   - 输入任务描述
   - 查看执行过程

## 功能介绍

### 1. 任务创建

在弹出窗口中描述你的任务：

**好的任务描述**：
- ✅ "在豆瓣上搜索评分8分以上的科幻电影"
- ✅ "帮我在GitHub找到react相关的UI组件库，按star排序"
- ✅ "在淘宝搜索蓝牙耳机，筛选500元以内的产品"

**不好的任务描述**：
- ❌ "帮我搜索" （目标不明确）
- ❌ "打开网页" （缺少具体网站和目的）

### 2. 任务执行

系统会自动：
1. 分析当前页面结构
2. 理解你的任务意图
3. 生成执行步骤
4. 逐步执行操作
5. 返回执行结果

### 3. 实时监控

在侧边栏中可以看到：
- 任务状态（规划中/执行中/已完成/失败）
- 每个步骤的执行进度
- 操作结果和提取的信息

## 任务示例

### 示例1：信息搜索

**任务**：查找最近的AI顶会截稿日期

**执行过程**：
1. 导航到 AI conference deadlines 网站
2. 查找包含 "deadline" 的元素
3. 提取前5个会议的截稿信息
4. 整理并返回结果

### 示例2：商品筛选

**任务**：在京东上找500-1000元的机械键盘

**执行过程**：
1. 在搜索框输入"机械键盘"
2. 点击搜索按钮
3. 设置价格筛选条件
4. 提取符合条件的商品信息

### 示例3：表单填写

**任务**：填写注册表单

**执行过程**：
1. 定位用户名输入框并填写
2. 定位邮箱输入框并填写
3. 定位密码输入框并填写
4. 勾选同意条款
5. 点击提交按钮

## 高级用法

### 自定义等待时间

如果页面加载较慢，可以在任务描述中说明：

```
在淘宝搜索商品，等待3秒加载完成后再筛选
```

### 多步骤任务

可以描述复杂的多步骤任务：

```
1. 在知乎搜索"人工智能"
2. 找到点赞最高的回答
3. 提取回答的主要观点
```

### 条件判断

AI会自动处理简单的条件判断：

```
如果页面有登录按钮，先点击登录，否则直接搜索
```

## 常见问题

### Q: 插件无法加载？

**A**: 检查以下几点：
- 是否开启了开发者模式
- manifest.json文件是否存在
- 是否选择了正确的dist目录

### Q: 任务执行失败？

**A**: 可能的原因：
- 页面结构发生变化
- 网络连接问题
- AI服务不可用
- 任务描述不够明确

### Q: 如何查看详细日志？

**A**: 
1. 打开 `chrome://extensions/`
2. 找到插件，点击"检查视图"
3. 在控制台中查看日志

### Q: 支持哪些网站？

**A**: 理论上支持所有网站，但效果可能因网站结构而异：
- ✅ 标准HTML结构的网站效果最好
- ⚠️ 单页应用(SPA)需要等待DOM渲染
- ❌ 使用Canvas/WebGL的页面无法操作

### Q: 如何提高任务成功率？

**A**: 建议：
- 任务描述要具体明确
- 对于复杂任务，分步骤执行
- 检查目标网站是否正常访问
- 确保AI服务连接正常

### Q: 插件会收集我的数据吗？

**A**: 不会。插件仅：
- 在本地浏览器执行操作
- 发送页面结构信息给AI（不含敏感数据）
- 不上传个人信息或浏览历史
